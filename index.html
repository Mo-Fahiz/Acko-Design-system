Acko Design System

The interactive playground is built and published by GitHub Actions (it is not the same as the files on the default branch root). Your published site must include a built index.html at the site entry — the workflow publishes apps/playground/dist for that.

If this page is all you see at your GitHub Pages URL: In the repo, go to Settings → Pages, and under Build and deployment choose GitHub Actions (not "Deploy from a branch" with only the raw repo). The workflow .github/workflows/deploy.yml must run successfully; then use the Visit site / environment URL from the workflow run.

Project site URL format: https://<owner>.github.io/<repository>/ — the repository name must match the Vite base in apps/playground/vite.config.ts (currently /Acko-Design-system/).
